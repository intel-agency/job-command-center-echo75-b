# Repository Summary: Job Command Center

## Overview

Job Command Center is a local-first LinkedIn job automation platform that attaches to an existing Chrome session via CDP. It prioritizes account safety by operating within an authenticated browser context rather than launching a fresh bot instance.

## Technology Stack

- **Language**: C# 12 / .NET 10
- **Orchestration**: .NET Aspire
- **UI**: Blazor Server + MudBlazor
- **Database**: PostgreSQL + Entity Framework Core
- **Automation**: Microsoft Playwright (CDP)
- **Observability**: OpenTelemetry, Serilog

## Build Commands

```bash
# Build solution
cd src/JobCommandCenter && dotnet build

# Run tests
dotnet test

# Run application (starts all services via Aspire)
dotnet run --project JobCommandCenter.AppHost
```

## Critical Constraints

1. **Harvester must run as host process** (not containerized) to access Chrome on port 9222
2. **No credential handling** - authentication piggybacks on existing Chrome session
3. **Human-mimicry delays** required to avoid detection
